Detailed Description

template<LatticeType LR, Compress COMPRESS, PerLevelOrder PLO>
class LatBuilder::StorageTraits< Storage< LR, EmbeddingType::MULTILEVEL, COMPRESS, PLO > >::Stride

Stride permutation.

The elements on each block are visited by applying a stride permutaion. The \(k^{\text{th}}\) level is the concatenation of the first \(k\) block. (more details in Storage)

For ordinary lattices: a Stride with parameter \(a \in \mathbb{N}\) is the mapping that maps an index \(i\) to \(a \times i \mod n\).
Consider the unpermuted vector \(\boldsymbol v = (v_1, \dots, v_n)\) for some positive integer \(n\). The \(j\)-th component of the vector with stride length \(a\) has value \(v_{j a \bmod n}\).

For polynomial lattices: A Stride with parameter \(q(z) \in \mathbb{Z}_2[z]\) is the mapping that maps a polynomial \(i(z)\) modulo \(P(z)\) with the polynomial \(h(z) = i(z)q(z) \mod P(z)\) .
Consider the unpermuted vector \(\boldsymbol v = (v_1, \dots, v_n) \sim (v_{1(z)}, \dots, v_{n(z)})\) for some positive integer \(n\), where \(j(z) = \sum a_lz^l\) if \(j = \sum a_l2^l\). Let \(P(z)\) be the modulus polynomial. The \(j\)-th component of the vector with stride parameter \(q(z) \in \mathbb{Z}_2[z]\) has value \(v_{j(z) q(z) \bmod P(z)}\).

If the per level order is inverse-cyclic: The stride length \(a\) can be identified to the \(i\)-th element \(g_i\) of the cyclic group \(\{ g_1, \dots, g_p \}\) integers (respectively polynomials) modulo \(n\) (respectively \(P(z)\)), with \(g_1 = 1\). Applying a stride with parameter \(g_i\) is equivalent to shifting the block elements by \( i \) positions, which leads to (square) circulant matrix-blocks if we go through the stride parameters (as column indexs) in the increasing order of \( i \). i.e in the cyclic order of the group.

A \(n \times n\) circulant matrix \(\boldsymbol C\) with elements \(C_{i,j}\) is completely defined in terms of its first column \(\boldsymbol c = (c_1,\dots,c_s)\), as \(C_{i,j} = c_{(i - j) \bmod n + 1}\).
